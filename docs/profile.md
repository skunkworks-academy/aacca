# Development Profile

## Current operating context

| Dimension | Current position |
|---|---|
| Current context | Property-business operations and digital-transformation planning; exact title to be confirmed. |
| Primary aspiration | Build stronger digital capability, optimise property processes and establish a scalable operating model. |
| Technology baseline | Digital payments, BI dashboards, accounting software, property-management software, online listings, Excel/Google Sheets, WhatsApp, cloud storage and CRM. |
| Data readiness | Information is stored in multiple systems but is not yet integrated into one governed operating view. |
| Internal IT capability | Basic user support; specialist architecture, integration and governance support will be required. |
| Delivery preference | Hybrid: improve and integrate existing tools first, then commission custom capability where justified. |
| Pilot interfaces | Owner, building manager, tenant, contractor, estate agent, executive dashboard and WhatsApp. |
| Initial investment context | R75,000–R150,000 pilot range documented in the discovery profile. |

## Strengths and strategic assets

| Strength | Development value |
|---|---|
| Operational domain exposure | Learning is grounded in real property workflows, stakeholders and commercial requirements. |
| Broad tool exposure | Existing accounting, CRM, BI, cloud, spreadsheet, listings and payment tools provide an integration foundation. |
| Transformation ambition | The intended operating model moves beyond isolated automation toward a connected and scalable service capability. |
| Commercial context | A pilot budget and possible revenue models allow learning to be tied to measurable business value. |
| Cross-functional orientation | Owner, tenant, contractor, manager, agent and executive needs create a strong stakeholder-leadership environment. |
| Pragmatic delivery preference | Improving the existing stack before custom development reduces cost and implementation risk. |

## Priority development gaps

| Gap | Required response |
|---|---|
| Integrated data model | Define authoritative sources, entity relationships, ownership, quality rules and a reporting layer before advanced automation. |
| Process discipline | Document current/future-state workflows, controls, hand-offs, service levels and exception paths. |
| Cloud and platform literacy | Build enough Azure, Microsoft 365 and Google Cloud knowledge to evaluate architecture, cost, risk and vendor proposals. |
| Low-code automation | Build practical Power Automate and Power Apps capability for controlled removal of manual work. |
| Analytics and measurement | Define reliable KPIs for occupancy, collections, arrears, maintenance, vacancy, response time and profitability. |
| Governance and security | Strengthen POPIA, access control, consent, auditability, payment control, backup and vendor-governance knowledge. |
| Portfolio evidence | Publish sanitised maps, dashboards, prototypes, decision records and outcome reports. |
| Leadership cadence | Create disciplined reviews, decision logs, adoption communication and accountable owners. |
