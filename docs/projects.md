# Applied Portfolio and Transformation Projects

## Project 1 — Property Operations Data Baseline

Create a governed dataset covering properties, units, tenants, leases, payments, arrears, vacancies, maintenance and key documents.

**Required evidence:** data dictionary, source map, ownership matrix, quality report, reconciliation results and privacy classification.

## Project 2 — Executive Performance Dashboard

Build an executive dashboard for occupancy, collections, arrears, vacancy days, maintenance cost, response time and portfolio performance.

**Required evidence:** metric definitions, data model, dashboard screenshots, refresh procedure, access model and management interpretation.

## Project 3 — Rental or Maintenance Workflow Automation

Automate one low-risk workflow such as maintenance triage, rental reminders, document approval or viewing requests.

**Required evidence:** current/future process maps, automation design, approval and exception paths, audit evidence, user test and measured improvement.

## Project 4 — PropTech Pilot Operating Model

Design the target operating model connecting existing systems, stakeholder portals, WhatsApp, reporting and governance.

**Required evidence:** architecture diagram, RACI, service catalogue, integration map, pilot plan, cost assumptions and risk register.

## Project 5 — AI-Assisted Operations Proof of Concept

Prototype a governed assistant for approved tenant enquiries, document classification or management reporting using approved or synthetic data only.

**Required evidence:** use-case definition, data boundary, prompt/response tests, human approval control, risk assessment and limitations statement.

## Minimum governance controls

- Use least-privilege access.
- Separate development and test environments from operational records.
- Do not place tenant, payment, identity or legal-document data into unapproved AI services.
- Define lawful purpose, ownership, retention, access and audit requirements before integration.
- Keep architecture and risk decision logs.
- Test exception paths, rollback, backup and manual fallback.
- Measure benefit against a documented baseline.
- Publish only sanitised evidence.
